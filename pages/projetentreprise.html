Ce stage de quatre semaines m'a permis de mieux comprendre le fonctionnement du CMS WordPress, d'utliser des plugins et d'apprendre à me servir de l'API de WordPress.

C'est quoi un CMS?

Un CMS Content Management System ou SGC en français ,système de gestion de contenu, regroupe des logiciels permettant de mettre en oeuvre des sites Web et l'un des plus renommés c'est WordPress qui est open-source et permet de construire un site web tres complet.


Grace à ses multiples extensions aussi bien pour la forme que pour les fonctionnalités on peut creer des pages bien structurées et l'on peut choisir un theme pour rendre le site harmonieux et stylé. On peut gérer le site facilement grace aux widgets petites applications qui s'integrent à la page web (dans WordPress c'est un module qui s'integre qui s'affiche dans le side bar).

Pourquoi utiliser un CMS quand on apprend le web-design? D'abord on peut integrer du code html , des thèmes personnalisés, mais WordPress peut faire gagner du temps quand on doit écrire beaucoup d'informations,et puis il a un systeme de rôles , par exemple vous avez le rôle d'administrateur,et vous attribuez le rôle de contributeur a un collègue qui pourra ainsi participer au site. De plus j'ai découvert l'utilisation de DIVI l'editeur de texte qui permet de structurer les pages d'un manière responsive c'est a dire adapter le contenu aussi bien à l'ordinateur qu'à la tablette.

C'est quoi une API?

une API signifie Application Programming Interface , pour WordPress on la désigne par API REST et permet de communiquer avec d'autres interfaces et applications. On peut ainsi transmettre des requêtes à travers l'URL et ces données sont sous format JSON.On peut aussi utiliser le logiciel Postman mais ou faire par exemple une application PHP pour introduire les données directement dans la base de données de WordPress. Par exemple pour les données Media on peut envoyer toutes les photos à la base de données.Certaines données sont publiques et d'autres sont privées et il faut s'identifier. Si on tape dans le navigateur http://nomdusite/wp-json on obtient les données publiques dans Chrome (on a besoin de l'extention jsonViewer).
