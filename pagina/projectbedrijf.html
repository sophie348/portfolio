Project in het bedrijf bij Opinaka

Deze stage van vier weken heeft me in staat gesteld om beter te begrijpen hoe het WordPress CMS werkt, om plugins te gebruiken en om de WordPress API te leren gebruiken..

Wat is een CMS?

Een CMS Content Management System omvat software voor het implementeren van websites en een van de meest bekende is WordPress, dat open-source is en waarmee u een website kunt bouwen.


Dankzij de meerdere extensies voor zowel vorm als functionaliteit, kunt u goed gestructureerde pagina's maken en kunt u een thema kiezen om de site harmonieus en stijlvol te maken. We kunnen de site gemakkelijk beheren dankzij de widgets kleine applicaties die in de webpagina zijn geïntegreerd (in WordPress is het een geïntegreerde module die wordt weergegeven in de zijbalk).

Waarom een ​​CMS gebruiken als we webdesign leren? Ten eerste kunnen we html-code, aangepaste thema's integreren, maar WordPress kan tijd besparen wanneer we veel informatie moeten schrijven, en dan heeft het een gebruikersrollen systeem, bijvoorbeeld heb je de rol van beheerder, en je wijst de rol van medewerker toe aan een collega die kan deelnemen aan de site. Daarnaast ontdekte ik het gebruik van DIVI, de teksteditor die het mogelijk maakt om de pagina's op een responsive manier te structureren, dat wil zeggen om de inhoud zowel aan te passen aan de computer als aan de tablet.

Wat is een API?

Een API staat voor Application Programming Interface, voor WordPress wordt het REST API genoemd en maakt communicatie met andere interfaces en applicaties mogelijk. We kunnen dus verzoeken doorsturen via de URL en deze gegevens zijn in JSON-formaat. We kunnen ook de Postman-software gebruiken of bijvoorbeeld een PHP-toepassing maken om de gegevens rechtstreeks in de WordPress-database in te voeren. Voor Mediagegevens kunt u bijvoorbeeld alle foto's naar de database sturen, sommige gegevens zijn openbaar en andere zijn privé en u moet zich legitimeren. Als we in de browser http://sitenaam /wp-json typen, krijgen we de openbare gegevens in Chrome (we hebben de jsonViewer-extensie nodig).
